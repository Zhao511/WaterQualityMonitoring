# 物模型对照检查报告

## 检查范围

对照用户提供的华为云 IoT 物模型规格，逐项验证：
- STM32 `iot_json.c`（JSON 序列化）
- STM32 `iot_service.c`（命令分发）
- ESP32 `thing_model.h`（物模型结构）
- ESP32 `main.cpp`（命令重构 + 响应转发）

## 服务 3：Water_status — 水质数据

### 属性（云端 5 个，全部可读可写）

| 云端属性 | 类型 | STM32 JSON | ESP32 thing_model | 判定 |
|---------|------|-----------|-------------------|:---:|

| `pH` | decimal | `"pH":%.2f` | `data["pH"] = ph` (float) | ✅ |
| `temp` | decimal | `"temp":%.1f` | `data["temp"] = temp` (float) | ✅ |
| `rfid` | string | `"rfid":"%s"` | `data["rfid"] = rfid` | ✅ |
| `gps` | string | `"gps":"%s"` | `data["gps"] = gps` | ✅ |

> ⚠️ **不一致**：ESP32 `thing_model.h` 第 107 行 `toCloudJson()` 中多了 `turbidity` 字段，但云端物模型无此属性。由于 ESP32 仅转发 STM32 JSON（不调用 `WaterStatus::toCloudJson()`），当前无影响。建议移除 `thing_model.h` 中的 `turbidity` 以保持一致性。

### 命令（5 个）

| 命令 | 云端参数 | STM32 解析 | STM32 响应 | 判定 |
|------|---------|-----------|-----------|:---:|
| `set_report_interval` | `interval` | ✅ `params.interval` (int, 1-3600) | `result` + `msg` | ✅ |
| `set_threshold` | `param, min, max` | ✅ 三个参数, pH大小写兼容 | `result` + `msg` | ✅ |
| `calibrate_sensor` | `sensor, mode, value` | ✅ 三个参数 | `result` + `msg` | ✅ |
| `led_control` | `color, mode` | ✅ 两个都解析 | `result` + `msg` | ✅ |
| `request_immediate_report` | 无 | ✅ 无参数读取 | `result` + `msg` ("reporting") | ✅ |

> ✅ **约束规则验证**：`request_immediate_report` 收到后调用 `IOT_Collect_All_Sensors()` + `IOT_Report_WaterStatus()`，符合"直接采集全量水质数据主动上报"规则。

| # | 规则 | 状态 |

| 1 | 属性按服务分组上报 | ✅ 4 服务各自 `service_id` + `properties` |

| 4 | decimal 用 float/double | ✅ `%.1f/%.2f/%.6f` |
| 5 | online 仅 true/false | ✅ `s->online ? "true" : "false"` |
| 6 | request_immediate_report → 全量采集上报 | ✅ `IOT_Collect_All_Sensors` + 立即上报 |
